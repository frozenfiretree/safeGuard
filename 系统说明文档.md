# SafeGuard 内网敏感文件检测与溯源系统说明文档

## 1. 文档目的

本文档用于说明当前 `safeGuard` 项目的实际系统形态、目录结构、启动方式、前后端交互、核心功能模块和后续维护注意事项。

当前系统已经不再使用旧版 `frontend/` 下的 React、Vite、Tailwind 前端工程。前端已改为后端直接托管的纯静态 WebUI：

- 前端入口：`server/webui/index.html`
- 技术形态：纯 HTML / CSS / JavaScript
- 图表能力：ECharts
- 背景动画：Canvas 粒子连线动画
- 认证方式：访问 WebUI 后先输入管理员 Token，验证成功后进入系统
- API 调用方式：浏览器端 `fetch()` 直接访问后端 `/api/v1/...`

## 2. 系统定位

SafeGuard 是一个面向内网环境的敏感文件检测与溯源系统，主要用于：

- 管理内网 Agent 终端
- 接收 Agent 上报的文件、事件、扫描状态和上传会话
- 对文件进行解析、OCR、规则检测和 LLM 复核
- 标记敏感文件并维护敏感档案
- 追踪敏感文件的版本变更、重命名、修改和删除
- 提供管理端 WebUI 展示系统态势、文件风险、事件日志、任务失败和系统配置
- 支持敏感文件归档下载，包括原文件、高亮文件和差异文件

系统当前采用“后端服务 + 后端托管静态 WebUI + Agent 客户端 + OCR 服务”的架构。

## 6. 启动方式

### 6.1 启动后端主服务

在项目根目录运行：

```bat
run_server.bat
```

该脚本会进入 `server/` 目录，并使用：

```bat
D:\anaconda3\envs\scanocr\python.exe
```

如果该 Python 不存在，则回退到系统 `python`。

默认启动地址：

```text
http://127.0.0.1:8000/
http://本机IP:8000/
```

默认开发 Token：

```text
dev-admin-token
```

`run_server.bat` 中如果未设置 `SAFEGUARD_ADMIN_TOKEN`，会自动设置：

```bat
set SAFEGUARD_ADMIN_TOKEN=dev-admin-token
```

后端启动命令本质是：

```bat
python -m uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

### 6.2 启动 OCR 服务

在项目根目录运行：

```bat
run_ocr_service.bat
```

OCR 服务默认监听：

```text
http://127.0.0.1:8010
```

后端通过环境变量 `SAFEGUARD_OCR_SERVICE_URL` 配置 OCR 服务地址，默认值为：

```text
http://127.0.0.1:8010
```

### 6.3 停止 OCR 服务

```bat
stop_ocr_service.bat
```

该脚本会查找监听 `8010` 端口的进程并结束。

## 8. 登录与认证机制

当前访问服务器地址后，首先进入 Token 登录页。

登录流程：

1. 浏览器打开 `http://127.0.0.1:8000/`
2. 页面显示 Token 登录界面
3. 用户输入管理员 Token
4. 前端调用：

```text
GET /api/v1/admin/agents
```

5. 如果返回 `200`，说明 Token 有效，进入控制台
6. 如果返回 `401`，停留在登录页并提示 Token 无效

前端支持以下 Token 输入形式：

```text
dev-admin-token
Bearer dev-admin-token
Authorization: Bearer dev-admin-token
X-Admin-Token: dev-admin-token
admin:password
```

普通 Token 会自动转换成请求头：

```http
Authorization: Bearer <token>
X-Admin-Token: <token>
```

后端管理员认证逻辑位于 `server/api.py`：

```python
def require_admin_auth(...)
```

后端支持：

- `Authorization: Bearer <SAFEGUARD_ADMIN_TOKEN>`
- `X-Admin-Token: <SAFEGUARD_ADMIN_TOKEN>`
- Basic Auth，前提是配置了 `SAFEGUARD_ADMIN_PASSWORD`

相关环境变量：

```text
SAFEGUARD_ADMIN_TOKEN
SAFEGUARD_ADMIN_USER
SAFEGUARD_ADMIN_PASSWORD
```

## 11. 检测与溯源流程

系统核心业务链路如下：

```text
Agent 注册
  ↓
Agent 拉取配置
  ↓
Agent 扫描本地文件
  ↓
文件初始化上传
  ↓
分片上传文件
  ↓
完成上传
  ↓
后端解析文件
  ↓
规则检测 / OCR 检测 / LLM 复核
  ↓
生成检测结果
  ↓
如果命中敏感内容，写入敏感档案
  ↓
后续文件改名、修改、删除事件进入版本时间线
  ↓
管理员在 WebUI 中查看、追踪、下载归档
```

敏感档案能力主要由：

```text
server/tracked_files.py
```

负责。

文件检测、解析和规则命中能力主要位于：

```text
server/detection/
server/services.py
```

```bat
run_ocr_service.bat
```

2. 启动后端：

```bat
run_server.bat
```

3. 打开浏览器：

```text
http://127.0.0.1:8000/
```

4. 输入 Token：

```text
dev-admin-token
```

5. 进入控制台。

### 19.2 查看管理页面

进入系统后可以依次查看：

```text
总览
Agent管理
资产发现
文件监控
敏感档案
检测规则
事件日志
上传会话
任务管理
运行状态
系统配置
```

### 19.3 下载敏感文件归档

1. 进入“敏感档案”
2. 选择左侧某个敏感文件
3. 在右侧版本时间线中选择对应版本
4. 点击：
   - 下载原文件
   - 下载高亮文件
   - 下载差异文件

## 20. 维护注意事项

### 20.1 不要恢复旧前端工程

当前系统前端基准是：

```text
server/webui/index.html
```

不要重新引入：

```text
frontend/
package.json
package-lock.json
node_modules/
deploy_frontend.bat
```

### 20.2 修改前端无需构建

修改 `server/webui/index.html` 后，刷新浏览器即可。

如果后端使用 `--reload`，后端会自动重载 Python 文件，但 HTML 文件通常只需要浏览器刷新。

### 20.3 修改接口时同步更新 WebUI

当前 WebUI 直接写死调用 `/api/v1` 下的真实接口。

如果后端接口路径、返回结构或认证方式改变，需要同步更新：

```text
server/webui/index.html
```

### 20.4 管理接口认证

当前 WebUI 入口有登录控制，但浏览器端控制不能替代后端接口保护。

已经受保护的接口：

```text
/api/v1/admin/*
```

建议后续也评估是否保护：

```text
/api/v1/rules
/api/v1/sensitive-files
```

### 20.5 生产环境 Token

开发 Token 是：

```text
dev-admin-token
```

生产环境必须替换为强随机 Token：

```text
SAFEGUARD_ADMIN_TOKEN=<strong-random-token>
```

不要在生产环境继续使用默认 Token。

### 20.6 OCR 服务依赖

如果“运行状态”页面显示 OCR 异常，需要检查：

- `run_ocr_service.bat` 是否已启动
- `8010` 端口是否监听
- OCR 模型是否存在
- Paddle / CUDA / CPU 环境是否正常
- `SAFEGUARD_OCR_SERVICE_URL` 是否配置正确

### 20.7 资产发现

资产发现页面依赖：

```text
GET /api/v1/admin/assets
POST /api/v1/admin/assets/refresh
```

刷新资产会调用后端本机探测与 ARP 缓存解析逻辑，并写入资产缓存。

### 20.8 失败任务

如果检测流程中出现失败任务，可以在“任务管理”页面重试。

失败任务接口：

```text
GET  /api/v1/admin/task-failures
POST /api/v1/admin/task-failures/{failure_id}/retry
```
